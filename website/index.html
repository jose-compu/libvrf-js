What is a VRF?

A Verifiable Random Function maps a secret key and an input to a unique pseudorandom output plus a proof. Only the key holder can produce the proof; anyone with the public key can verify it. VRFs are used in consensus, lotteries, DNSSEC, and randomness beacons.

Features

  • RFC 9381 RSA-FDH and RSA-PSS-NOSALT (interop with Microsoft libvrf)
  • Simplified EC VRF on P-256 (browser + Node; not RFC wire-compatible)
  • TypeScript types, VRFError, isRFC9381Compliant()
  • RSA cross-vectors, fast CI matrix, browser bundle

Installation

npm install libvrf

Quick start (Node.js)

import { VRF, VRFType } from 'libvrf';

const type = VRFType.RSA_FDH_VRF_RSA2048_SHA256;
const sk = VRF.create(type)!;
const pk = sk.getPublicKey()!;

const input = new TextEncoder().encode('hello');
const proof = sk.getVRFProof(input)!;
const [ok, vrfValue] = pk.verifyVRFProof(input, proof);

console.log('verified:', ok);
console.log('vrf value:', Buffer.from(vrfValue).toString('hex'));

Browser (EC VRF only)

const sk = await libvrf.VRF.createAsync(libvrf.VRFType.EC_VRF_P256_SHA256_TAI);
const pk = await sk.getPublicKeyAsync();
const input = new TextEncoder().encode('hello');
const proof = await sk.getVRFProofAsync(input);
const [ok, value] = await pk.verifyVRFProofAsync(input, proof);
