Concept

What is a Verifiable Random Function?

A VRF is a function β = fSK(α) where only someone who knows the secret key SK can compute β and a proof π. Anyone with the public key PK can check π quickly and recover the same β.

🔑

Unique & pseudorandom

β looks random to anyone without π. Same (SK, α) always gives the same β—useful for deterministic lotteries and leader election.

📄

Compact proof π

The prover sends π with α. Verifiers do not need SK and do not redo the expensive derivation.

✓

Public verify

Validators, contracts, or clients confirm correctness with PK only—no trust in the prover’s hardware.

Who holds what?

Prover — SK, runs getVRFProof
Verifier — PK, runs verifyVRFProof
Everyone — agrees on input α

libvrf

This library

TypeScript port of Microsoft libvrf — RSA in Node.js, EC in browsers via WebCrypto.

RSA-FDH

RFC 9381 full-domain hash. 2048–4096 bit keys, SHA-256/384/512. Cross-tested vs C++ vectors.

RSA-PSS-NOSALT

RFC 9381 PSS variant without salt. Same key sizes; interop with libvrf wire format.

EC P-256 (browser)

Simplified EC VRF for demos & apps. Use async APIs; not RFC ECVRF wire format.

Production: Prefer RSA suites on Node for RFC 9381 interop. Use the live demo to learn SK / PK / π / β, then deploy RSA in your backend.

Developers

Quick start

npm install libvrf
import { VRF, VRFType } from 'libvrf';

const sk = VRF.create(VRFType.RSA_FDH_VRF_RSA2048_SHA256)!;
const pk = sk.getPublicKey()!;
const alpha = new TextEncoder().encode('hello');

const proof = sk.getVRFProof(alpha)!;
const [ok, beta] = pk.verifyVRFProof(alpha, proof);
console.log(ok, Buffer.from(beta).toString('hex'));
